第一部分，角色与硬性约束：用一句话锁定模型身份，并明确写入「你只能输出合法的 JSON 对象，禁止输出任何解释文字、禁止添加 Markdown 代码围栏、禁止前后缀」；

第三部分，输出模板：给出一份可直接复制的 JSON 骨架（用花括号占位符表示待填内容），字段顺序与 Schema 一致；
第四部分，容错与重试策略：当模型偶尔输出非纯 JSON（如带代码围栏或多余说明）时，给出后处理逻辑（定位第一个左花括号与最后一个右花括号截取、去除代码围栏、空值兜底）；

第六部分，常见陷阱提示：列出温度参数过高、系统提示与用户约束冲突、字段过多导致遗漏、布尔值被写成字符串等典型问题，并给出规避办法。
所有说明必须用中文，示例要真实可运行，不要用空洞的占位话术凑数。
